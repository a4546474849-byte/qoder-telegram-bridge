# Cron Jobs Format for Qoder CLI

To create a scheduled job, edit `C:\Qoder_CLI\qoder-telegram-bridge\cron.json`

## Structure

```json
{
  "version": 1,
  "jobs": [
    {
      "id": "unique-uuid-here",
      "name": "Job name",
      "enabled": true,
      "createdAtMs": 1700000000000,
      "updatedAtMs": 1700000000000,
      "schedule": { "kind": "every", "everyMs": 1800000 },
      "sessionTarget": "isolated",
      "wakeMode": "now",
      "payload": {
        "kind": "agentTurn",
        "message": "Check email and summarize new messages",
        "timeoutSeconds": 300
      },
      "delivery": {
        "mode": "announce",
        "channel": "telegram",
        "to": "382401183"
      },
      "state": {}
    }
  ]
}
```

## Schedule Types

### 1. Every N minutes (recurring interval)
```json
"schedule": { "kind": "every", "everyMs": 1800000 }
```
`everyMs` in milliseconds. 30 min = 1800000, 1 hour = 3600000

### 2. Cron expression
```json
"schedule": { "kind": "cron", "expr": "0 9 * * *", "tz": "UTC" }
```
Standard cron: minute hour day month dayOfWeek

### 3. One-shot (specific time)
```json
"schedule": { "kind": "at", "at": "2026-05-12T10:00:00.000Z" }
```

## Payload

Always `agentTurn` for Qoder jobs:
```json
"payload": {
  "kind": "agentTurn",
  "message": "What you want Qoder to do",
  "timeoutSeconds": 300
}
```

## Delivery

Sends result to Telegram:
```json
"delivery": {
  "mode": "announce",
  "channel": "telegram",
  "to": "382401183"
}
```

## Example: Add a job

Generate UUID with: `node -e "console.log(require('crypto').randomUUID())"`

Add to `jobs` array in cron.json, save file. The bridge auto-reloads within 1 second.
